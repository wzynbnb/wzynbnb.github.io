DOM事件流

事件流描述的是从页面中接收事件的顺序

比如，当点击了一个注册了onclick事件的div时

DOM事件流进入捕获阶段，从最外层开始，Document收到点击事件，下面html收到点击事件，下面body收到点击事件，下面div收到点击事件

DOM事件流进入当前目标阶段，div对事件进行处理

DOM事件流进入冒泡阶段，向上层传递该事件，看看body有没有onclick事件，有就也触发，再看html、document，一层一层向上

可以通过 addEventListener(type,listener[,useCapture]) 的第三个参数 useCapture 来设置绑定的事件是在事件捕获阶段调用处理程序还是在事件冒泡阶段调用处理程序。

注意事项
  1. JS代码只能执行捕获、冒泡阶段其中之一
  2. onclick、attachEvent 这两种方式绑定事件，只能得到冒泡阶段，也就是从最底下往上依次执行处理程序，addEventListener 可以得到捕获阶段
  3. 在开发中很少使用捕获阶段，一般更关注冒泡阶段
  4. 一些事件比如 onblur、onfocus、onmouseenter、onmouseleave等是没有冒泡的

举例：

一个父盒子中有一个子盒子

如果给父盒子绑定mouseover事件，则鼠标进入父盒子会触发，鼠标进入到子盒子时，父盒子的mouseover绑定的事件也会触发

如果给父盒子绑定mouseenter事件，则鼠标进入父盒子会触发，进入子盒子不会再触发父盒子的mouseenter绑定事件

原理：

mouseover是会冒泡的，当鼠标进入子盒子，虽然子盒子没有绑定mouseover对应的事件，但是它会把这个mouseover事件向上冒泡，父盒子还是会收到mouseover事件，所以会再次被触发

mouseleave

与mouseenter相对应，mouseleave是鼠标离开元素触发，也不会冒泡

BOM

概述

BOM，浏览器对象模型，核心对象是window，提供了独立于内容而与浏览器窗口进行交互的对象。BOM缺乏标准，兼容性不是很好

BOM控制浏览器的，返回上一页，回到刚才页，刷新，滚动条，浏览器窗口大小调整等等与浏览器相关的事件

BOM与DOM

DOM操控的是浏览器页面部分

DOM操控的是整个浏览器部分

BOM的构成

DOM其实是位于BOM下级的，DOM获取元素对象的语句可以这样写

window对象是浏览器的顶级对象

  1. window是JS访问浏览器窗口的一个接口
  2. window是一个全局对象，定义在全局作用域中的变量、函数、都会变成window对象的属性和方法
  3. 在使用各种变量、函数时，不必要加上前面的 window.
  4. window下有一个特殊属性 name ，自己声明变量时不要使用 name 作为变量名

JS执行机制

异步和同步

同步：代码按顺序执行

异步：不同位置的代码可以同时执行

同步任务都是在主线程上执行，形成一个执行栈

异步任务，有三种类型：

  1. 普通事件：click、resize等
  2. 资源加载：load、error等
  3. 定时器：包括setInterval、setTimeout等

异步任务相关的回调函数添加到任务队列（消息队列）中

执行栈、任务队列，是两个排队的地方，同步任务在执行栈中先执行完，然后再去执行任务队列中的异步任务

执行顺序：先同步、再异步

当按照顺序执行代码时，碰到了异步任务，会JS会提交给对应的异步进程处理，处理完毕后（这里的处理完并不是真正的完成，而是能完成的都完成了，其余的等到主线程执行完毕后等主线程来完成），推入任务队列，等待主线程执行完毕，主线程会查询任务队列，并找出一个任务，推到主线程处理，主线程执行完毕后查询任务队列是不断重复的过程（事件循环）。

比如onclick事件，按顺序执行到他之后，主线程把他交给对应的异步进程处理，直到用户有了对应的点击操作，异步处理进程再把它推入任务队列，等待主进程执行完毕来执行它

用offset与style获取元素对象属性的区别

offset可以得到任意样式表的样式值，style只能得到行内样式表的样式值

offset获得的是纯数值没有单位，style获得的是有单位的字符串

offsetWidth包含padding border width，style获得的只有width，不包含padding border

offsetWidth等属性是只读的，不能赋值，而通过style可以读取也可以赋值

结论：

只获取元素大小位置等信息，使用offset

需要操控样式，使用style
